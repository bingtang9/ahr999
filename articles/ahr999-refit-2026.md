最近这波行情，身边好几个朋友问我：**"ahr999 怎么一直显示抄底啊？"**

大家都能感觉到这个指标这两年不太对劲——BTC 这轮周期已经好几次摸到历史新高附近，按市场感觉根本算不上"低位"，但 ahr999 的读数却经常落在 0.4 上下甚至直接跳进"抄底"区间。**指标给出的位置，比市场实际的位置明显偏乐观**。那问题出在哪？

## 一、先复习一下 ahr999 是什么

这个指标 2018 年由微博博主 @ahr999 提出，本质是一个"囤币辅助信号"：

```
ahr999 = (现价 / 200日定投成本) × (现价 / 长期公允价)
```

**200 日定投成本**：过去 200 天 BTC 的平均价——代表"你如果一直定投，平均持仓成本大概是多少"。

**长期公允价**：BTC "理论上应该值多少"——由一条拟合曲线给出。

两个数字**同时小于 1**，说明现在的价格比两个维度都便宜，典型抄底区；**同时大于 1**，说明两头都贵了，该停手。

原作者定义的三段阈值很经典：

| 区间 | 对应动作 |
|---|---|
| ahr999 < 0.45 | **抄底**：显著低估 |
| 0.45 – 1.2 | **定投**：常态区间 |
| ahr999 > 1.2 | **停止定投**：相对高估 |

---

## 二、问题出在那条"公允价"曲线上

公允价的算法是这样：

```
公允价 = 10 ^ ( slope × log10(币龄天数) − c )
```

这里的 `slope` 和 `c` 两个数字**不是数学推出来的**，是作者把 BTC 历史价格画在对数坐标上，**用一条直线去拟合**得出来的。

2018 年作者做这件事的时候，BTC 只有 **10 年历史**，他得到的参数是：

```
slope = 5.84
c     = 17.01
```

BTC 的价格在对数图上确实像一条直线，所以这套公式在 2018 年跑得很好。**但是**——你用 10 年数据画的直线，**一直外推到 17 年**，末端就会偏离真实价格轨迹。外推越远，偏得越厉害。

这就是为什么大家感觉"ahr999 失效"的根本原因：**不是指标错了，是用的那条线太老了**。

## 三、我自己拉数据跑了一遍

口说无凭，上数据。

我从 **Coin Metrics** 公开 API 拉了 **2010-07-18 到 2026-04-24** 的全部 **5,760 天** BTC 每日收盘价，用标准最小二乘法重新拟合了一条直线。完整脚本都在项目里（文末有仓库链接），任何人都可以复现。

**重新拟合的结果**：

```
slope = 5.6793     （旧值 5.84）
c     = 16.4746    （旧值 17.01）
R²    = 0.9615
```

**R² = 0.96** 用大白话翻译就是：**这条新直线解释了 BTC 对数价格 96% 的波动**，剩下 4% 是噪声。模型本身高度可信。

### 新旧参数差多少？用今天对比

把今天（币龄 6,319 天）代入两组参数：

| 参数组 | 今天 BTC 公允价 |
|---|---|
| 旧 (5.84 / 17.01) | **$153,386** |
| 新 (5.68 / 16.47) | **$126,982** |

差 **$26,000**，大约 **21%**。

这 21% 直接灌到 ahr999 的分母里，让整个指标被系统性压低：

| 参数 | 今日 ahr999 | 显示区间 |
|---|---|---|
| 旧 | **≈ 0.37** | 抄底 🟢 |
| 新 | **≈ 0.56** | 定投 🟡 |

**这一下就破案了**：不是 BTC 真的在抄底区间待了两年，是用的那条老线把指标往下压了。**用重拟后的参数看，市场其实早几个月前就从"抄底"回到"定投"区间了。**

### 和 2018 年那时候对比一下

我还刻意做了第二次拟合，只用 **2018 年之前** 的数据（2,724 天），还原一下原作者当年的情况：

```
slope = 5.7277
R²    = 0.9085
```

R² 只有 **0.91**，明显比现在的 0.96 低。不是原作者拟得差——是**当时样本就那么多，早期 BTC 价格又特别噪**，能拟到 0.91 已经很漂亮。

今天多了 8 年数据之后，**参数自然就稳定了、R² 也高了**。这是数据量带来的必然进化。

## 四、做了个实时页面

我把上面的结论做成一个实时监控页面：

> **https://bingtang9.github.io/ahr999/**

> *【截图位 1：页面顶部——BTC 价格、ahr999 当前值、区间徽章、历史分布】*

**主要功能**：

- **实时更新**：每 60 秒拉一次 Binance 最新 BTC 价格，今日 ahr999 秒级变化
- **完整历史**：曲线从 2011 年起（200 日均线预热所需），牛熊周期全覆盖
- **三条线**：ahr999（蓝）+ BTC 价格（橙）+ 200 日定投成本（灰白）
- **两条阈值线**：抄底线 0.45 和定投线 1.2，一眼知道自己在哪个区间
- **时间轴拖动**：下方 brush 可以任意选段，或快捷按钮 `1M / 3M / 6M / 1Y / 3Y / 全部`
- **历史分布**：告诉你 BTC 这些年有多少天在"抄底"、多少天在"定投"、多少天在"停止定投"——**你会发现"抄底"的天数其实非常珍贵**
- **参数自更新**：脚本每 90 天自动重拟一次，**永远不会像原版那样过期**
- **PWA**：手机加到桌面后像 app 一样用，弱网也能看历史

## 五、写在最后

说点不那么严肃的话。

这次"自己跑一遍"的过程，其实印证了一个老观念：**任何基于历史数据的模型都有保质期**。ahr999 这种指标**本身没错**，错的是时间——用 17 年前不存在 BTC 时的数据去校准 2026 年万亿美金资产的行情，逻辑上就不合理。

**"每隔几年重拟一次"应该成为使用这类指标的默认动作**。我把这件事做进了脚本：每 90 天自动重跑一次 OLS 回归，新参数自动写入配置文件。以后不管多少年，这个页面上的数字都会跟着 BTC 的成长"自动校准"。
